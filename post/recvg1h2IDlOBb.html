量大管饱！ 用 Coxy 解锁GitHub Copilot的全部潜力

每个月GitHub发给你一堆 Copilot 额度，大部分人用掉的比例少得可怜。

而作为10刀及以上订阅用户，官方更是提供了几乎无限的GPT-4o等免费使用。

只是用来Copilot估计怎么也不完，我们可以将这个服务代理出来给其它的Chat或者OpenClaw来使用。

我使用的是Coxy 是一个本地代理服务，它把 GitHub Copilot 的请求协议翻译成标准的OpenAI 兼容 API。

启动之后，你本地会多出一个 http://localhost:3000/api 的接口，任何支持 OpenAI API 的工具都可以接进来，包括 Open WebUI、LLM CLI，或者你自己写的脚本。

用法极简。不想装 Node.js 的话，一行 Docker 命令搞定：

docker run -p 3000:3000 ghcr.io/coxy-proxy/coxy:latest

然后浏览器打开 localhost:3000，用 GitHub 账号登录，生成一个 token，设为默认。

之后所有请求不需要带 Authorization 头，直接打就行。

它背后的机制其实挺有意思。

Copilot 的 VS Code 插件本身就是通过 @github/copilot-language-server 这个包跟 GitHub 服务器通信的。

Coxy 用的是同一个包，相当于在 VS Code 之外起了一个相同的语言服务器，然后在外面包了一层 OpenAI 兼容的接口。

目前支持的模型包括 gpt-4.1、gpt-4o，以及 GPT-5 mini（需要在 GitHub 账号设置里手动开启，路径是 github.com/settings/copilot/features）。

如果你有多个 GitHub 账号，每个账号都有独立的免费额度，Coxy 支持多 token 管理，可以手动切换。

这对于有多账号习惯的人来说相当实用，不然那些额度就真的是纯浪费。

数据存在本地 SQLite 里，用 Docker 挂载的话记得把 db 文件 mount 出来，不然重启就没了。

安装一行搞定

npm install -g @github/copilot
# 或者
curl -fsSL https://gh.io/copilot-install | bash

装完 cd 进项目，运行 copilot，然后 /login 认证 GitHub 账号。

第一件事建议跑 /init，会根据你的项目生成专属指令文件。

大部分用法和Claude Code一样。

常用的操作

Shift+Tab 在三个模式间切换。

Chat 模式是普通问答，不执行操作。

Agent 模式是默认的，规划任务、改文件、跑命令，每一步让你确认。

Autopilot 模式是放手跑到完成，适合任务边界清晰的场景。

Subagent 并行

/fleet 命令是 Copilot CLI 里最有意思的设计。

它会把一个大任务拆解成独立子任务，同时派出多个 subagent 并行执行，每个 subagent 有独立的 context window，底层用 SQLite 维护任务依赖图。

/fleet 给 src/services/ 下每个 service 加单元测试

还可以在 prompt 里指定特定 subagent 用哪个模型：

Use Claude Opus 4.5, to analyze the security issues
Use @test-writer to create unit tests

效果是跳过所有工具调用的逐条确认，信任 agent 自由执行。加 --max-autopilot-continues 30 设上限防止死循环。

省配额的方式

Copilot CLI 的计费单位是 premium request，不同模型倍率差很多。

GPT-4.1 和 GPT-5 mini 是免费的，不消耗配额，日常任务够用。

Claude Haiku 4.5 倍率 0.33x，Sonnet 居中，Opus 是 3x。用 /fleet 要注意，每个 subagent 独立计费，5 个 subagent 跑一个任务，消耗可能是单 agent 的好几倍。

Skills、Hooks、MCP、Autopilot、自定义 agent，概念几乎一一对应。核心差异是 Copilot CLI 原生打通了 GitHub 的 issues、PR、repo 上下文，对本来就在 GitHub 上工作的团队几乎零迁移成本。

模型选择上比 Claude Code 更灵活，可以在 Claude 和 GPT 系列之间切换。

如果你的工作流已经在 GitHub 上，值得装一个试试。免费计划也能用，只是 premium request 配额更少。
