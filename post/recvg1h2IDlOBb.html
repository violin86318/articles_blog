量大管饱！ 用 Coxy 解锁GitHub Copilot的全部潜力

一句话总结

Coxy通过本地代理将GitHub Copilot协议翻译为OpenAI兼容API，解锁其全部潜力并支持多账号管理。

核心摘要

核心工具：Coxy本地代理 功能：将GitHub Copilot私有协议转为标准OpenAI API（localhost:3000/api），兼容Open WebUI等第三方工具。 部署：一行Docker命令启动（docker run -p 3000:3000 ghcr.io/coxy-proxy/coxy:latest），免Node.js依赖。 机制：复用Copilot官方语言服务器包（@github/copilot-language-server），外部封装API接口。 关键特性： 1. 多模型支持：直接调用gpt-4.1、gpt-4o及需手动开启的GPT-5 mini模型； 2. 多账号管理：支持绑定多个GitHub账号独立使用免费额度，数据本地SQLite存储（Docker需挂载防丢失）； 3. 免鉴权简化：设置默认token后无需Authorization头。 延伸工具：Copilot CLI提供命令行代码编写/部署能力。两者结合可最大化释放Copilot潜力，尤其适合多账号用户避免额度浪费。

正文整理

文章标题为《量大管饱！ 用 Coxy 解锁GitHub Copilot的全部潜力》，作者为字节笔记本。文章主要介绍了如何使用Coxy这个本地代理服务来解锁GitHub Copilot的全部潜力。

Coxy是一个本地代理服务，它将GitHub Copilot的请求协议翻译成标准的OpenAI兼容API。使用Coxy后，你本地会多出一个http://localhost:3000/api的接口，任何支持OpenAI API的工具都可以接进来，包括Open WebUI、LLM CLI，或者你自己写的脚本。

使用Coxy的步骤很简单。如果你不想安装Node.js，可以使用一行Docker命令搞定：docker run -p 3000:3000 ghcr.io/coxy-proxy/coxy:latest。然后，在浏览器中打开localhost:3000，使用GitHub账号登录，生成一个token，并将其设置为默认。之后，所有请求都不需要带Authorization头，直接发送即可。

Coxy背后的机制很有意思。GitHub Copilot的VS Code插件通过@github/copilot-language-server这个包与GitHub服务器通信。Coxy使用的是同一个包，相当于在VS Code之外启动了一个相同的语言服务器，然后在外面包了一层OpenAI兼容的接口。目前支持的模型包括gpt-4.1、gpt-4o，以及GPT-5 mini（需要在GitHub账号设置里手动开启）。

如果你有多个GitHub账号，每个账号都有独立的免费额度，Coxy支持多token管理，可以手动切换。这对于有多账号习惯的人来说非常实用，不然那些额度就真的是纯浪费。数据存储在本地SQLite中，如果你使用Docker挂载，记得把db文件mount出来，否则重启后数据会丢失。

此外，文章还介绍了Copilot CLI的使用。Copilot CLI是Copilot早自己的CLI工具，安装非常简单。安装完成后，你可以使用它来进行代码编写、部署和运维等工作。

总的来说，Coxy和Copilot CLI都是非常实用的工具，可以帮助你更好地使用GitHub Copilot，解锁其全部潜力。如果你已经在GitHub上工作，值得尝试一下这两个工具。
