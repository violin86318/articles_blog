全世界第一个音频Infra上线，让调用音乐像调用模型一样简单

过去1年，我和43Music团队创作了10万首歌，过去一个月，我开发了一个Audio Infra，把这些音乐全部开源了。

这就是Audiolib.ai，全世界第一个服务于开发者的音频基建层。

你有没有想过，为什么全世界99%的App和网站是没有声音的？

因为这个事在Ai时代来临之前门槛太高了，去哪里搞曲库? 版权怎么办? 授权一年多少钱? CDN 谁来扛? 要不要自己搭一个播放器? 要不要做一个后台让运营选歌? 这事干完得多少钱、多少人、多少天? 

而在Ai爆发之后，这事也没有变的很简单，你可以很轻松的用Ai做几首歌，但是你有没有人、时间和专业能力做出足够多的高质量音乐？对多数开发者来说，这不是自己的业务专长所在，想到这里就放弃了。

不是用户不需要,是开发者搞不动。

Audiolib 就是为了让你搞得动而做的。

它是什么: 音频版的OpenRouter

你已经知道怎么调一个大模型了。

POST /v1/chat/completionsAuthorization: Bearer sk_xxxxx

Audiolib 让你用一模一样的姿势调一首音乐。

POST https://api.audiolib.ai/v1/audioAuthorization: Bearer alp_xxxxxContent-Type: application/json{"library":"audio.focus"}

没有 SDK, 没有播放器, 没有后台, 没有版权谈判, 没有曲库管理, 只需一个Key一个曲库名称，直接在你的产品里播放音乐。

把它理解成:硅基流动给你模型, Audiolib给你音乐。

模型按 token 计费, 音乐按唤起计费，每一次成功调用 = 1首歌。

每调用一次, 完整播放一首歌, 使用成本最低仅0.0003美元。

播放10,000首歌3美元。

让用户在你的产品里全天候听音乐, 你一个月可能花不到10美元。

像烧token一样烧音乐，但你完全不需要为"用户会不会听太多"焦虑——这个单价就是为了让你不焦虑地把音乐铺进产品里。

我们做了什么准备工作,你才能这么便宜地用

Audiolib不是一个聚合器, 不是版权代理, 不是 Suno 的中转。

是我们团队自己用一年多时间生成、筛选、分类、标注的 10 万+ 首各种类型的音乐, 按场景和风格切分成几十个曲库。

每一个曲库都有清晰的定位:

功能向(audio.focus / audio.relax / audio.sleep / audio.workout / audio.meditation / audio.study / audio.energy / audio.running / audio.emotional / audio.flow / audio.healing / audio.background...)

风格向(audio.lofi / audio.ambient / audio.house / audio.pop / audio.rock / audio.electronic / audio.chinese-pop / audio.mandarin-ballad / audio.disco / audio.trance / audio.techno / audio.dubstep / audio.new-age / audio.world-music / audio.jazz...)

每个曲库从数百到上千首音乐作品，每次 call 随机返回一首, 支持"无限播放"的效果。

等于我把自己创作的 10 万首音乐, 以 API 形式开源给了所有开发者。

怎么开始？

  1. 去audiolib.ai 注册,免费拿 300 calls/月

  2. Dashboard 生成 API Key

  3. 选一个曲库, 试听是否符合你的需求

  4. 把一行 fetch 放进你的产品

整个过程不超过五分钟。
