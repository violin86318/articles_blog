装了七八回才摸透——用 Antigravity 安装 OpenClaw 完整教案

| 亲测版 · 只写用过的

OpenClaw 是什么？

OpenClaw（龙虾） 是一个开源的自托管 AI Agent 网关，让你在自己电脑上跑一个真正能干活的 AI 助手——写代码、管文件、执行任务，24 小时在线。

跟普通 AI 聊天工具的本质区别是：它是员工，不是客服。

这篇写给谁看

✅ 适合的人，同时满足两点：

1. 能自己配网络环境

2. 能购买 Gemini Pro（闲鱼买学生邮箱版，约 80 元，以前 30 元）

⚠️ 暂时不适合的人：

国内没有网络配置能力的，我不做强烈推荐。这事背后取决于底模能力——目前国内智谱和 Kimi 相对优秀，Kimi 甚至更好。可以去试，但别抱太大希望，能比以前的 AI 多一些干活能力，但不是自媒体博主说的那种神器。

腾讯和 360 不建议——这类厂商会在后台做手脚让你多消耗资源，踩过，不推荐。

准备工作

硬件：找一台闲置旧电脑

Mac mini、MacBook Air、MacBook Pro 都行。

⚠️ 关键：登录时用游客账号，不要用你的主账号

>

原因：多层隔离。OpenClaw 的安全性是它最薄弱的环节，物理隔离是最稳的保障。

系统要求：

🔹 Node.js 22+（终端运行 node --version 检查版本）

🔹 网络环境已配好

账号：Gemini Pro

闲鱼搜「Gemini Pro 学生邮箱」，买评分高的，约 80 元一个。

第一步：获取 Google AI Studio API Key

OpenClaw 安装时会问你 API Key，提前准备好。

地址：aistudio.google.com

步骤：

1. 打开 [aistudio.google.com](https://aistudio.google.com)，用 Google 账户登录

2. 左侧菜单点击 「Get API Key」 → 「Create API Key」

3. 给密钥取个名字（比如 openclaw-key）

4. 选择关联的 Google Cloud 项目（新用户会自动创建默认项目）

5. 复制 API Key，妥善保存

可用模型（免费层）：

🔹 Gemini 2.0 Flash（100 万 token 上下文）

🔹 Gemini 1.5 Pro（200 万 token 上下文）

💡 这个 API Key 就是你 OpenClaw 运行的主力模型来源，稳定靠谱。

第二步：下载安装 Antigravity

Antigravity 是谷歌的 AI 原生 IDE，完全免费。它在这里扮演三个角色：安装 OpenClaw、日常调试、问题检查。

下载地址：antigravity.im/download（支持 Mac / Windows / Linux）

安装完成后打开 Antigravity，进入下一步。

第三步：安装 OpenClaw

方式一：通过 Antigravity 安装（推荐）

在 Antigravity 内选择模型为 Claude Opus 4.6，然后直接告诉它：

「帮我安装 OpenClaw」

Antigravity 会自动处理安装流程，并在需要时引导你输入 API Key。

方式二：终端命令安装

npm install -g openclaw@latest

安装完成后启动配置向导：

openclaw onboard --install-daemon

向导会问你：

🔹 选择模型提供商 → 选 Google Gemini

🔹 输入 API Key → 粘贴第一步获取的 AI Studio Key

🔹 完成 Gateway 配置

验证安装成功： 打开浏览器访问 http://127.0.0.1:18789/，能看到对话界面即成功。

第四步：配置 Gemini 为主要模型

安装向导中配置，或后续手动编辑：

openclaw config edit

配置文件示例：

{   "providers": {     "google": {       "provider": "google",       "model": "gemini-2.0-flash",       "apiKey": "${GOOGLE_API_KEY}"     }   } }

💡 API Key 用环境变量引用，不要直接写进文件，安全。

第五步：部署方式

本地部署 ✅　　云端部署 ❌

"对于小白来说，本地部署比云端部署好，云端部署坑特别多。"

腾讯云踩过，坑很多。直接本地，稳。

装完之后，第一件事：写配置文件

这步 99% 的人会跳过，但最重要。

OpenClaw 裸奔状态安全性极差，会动不动莫名死掉。你需要给它写一套行为手册，放在 ~/.openclaw/workspace/ 目录下。

必须配置的 4 个核心文件：

| 文件名 | 作用 |

|--------|------|

| SOUL.md | 龙虾的核心人格与行为准则，它是谁、怎么说话、怎么做事 |

| IDENTITY.md | 身份配置，权限边界，能做什么不能做什么 |

| TOOLS.md | 声明它可以调用哪些工具和技能 |

| SHIELD.md | 安全策略，如何排查病毒、防止恶意指令 |

基础版本（可直接使用）：

SOUL.md

# Soul  你是一个高效、安全、专注的 AI 助手。 - 优先完成用户明确交代的任务 - 不主动执行未被授权的操作 - 遇到不确定的情况，先问再做 - 用简洁中文回复，除非用户要求英文

IDENTITY.md

# Identity  - 名字：龙虾助手 - 主人：[你的名字] - 工作语言：中文优先 - 权限级别：本地文件读写、网络搜索、代码执行 - 禁止：删除系统文件、访问未授权目录、对外发送敏感信息

TOOLS.md

# Tools  ## 已启用工具 - 文件读写（本地目录） - 网络搜索 - 代码执行（沙盒环境） - 日志查看与分析  ## 禁用工具 - 系统级命令（rm -rf 等危险操作）

SHIELD.md

# Shield  ## 安全规则 - 执行任何删除操作前必须二次确认 - 检测到异常进程时立即暂停并报告 - 禁止向外部发送 API Key 或密码 - 每次启动自检：扫描配置文件是否被篡改  ## 病毒排查 若行为异常，运行：`openclaw diagnose --full`

💡 更好的方式：让 Antigravity 帮你写

>

打开 Antigravity，选 Claude Opus 4.6，告诉它：

>

「帮我为 OpenClaw 生成 SOUL.md、IDENTITY.md、TOOLS.md、SHIELD.md 四个配置文件，放到 ~/.openclaw/workspace/ 目录下，要有安全性和病毒排查能力。」

>

Opus 4.6 会根据你的需求写好并自动放进去，比手动配置更精准。

关于「虾会不断死」这件事

说实话：你的虾一定会死，而且会死很多次。

没有编程背景、不会看日志——这不是问题，Antigravity 就是来干这件事的。每次虾死了，Antigravity 会自动跑到后台分析原因，帮你查日志、找问题、自动修复。你叫它去网上查方案，它也完全可以。

这是我装了七八回最终摸透的组合——有了 Antigravity 当主管，虾死了不慌。

备选方案：学会其中一个就够了

Antigravity 是推荐方案，但不是唯一选择。下面三个工具都可以充当你龙虾的坚实后盾，学会其中一个，就能无数次把虾救活：

| 工具 | 类型 | 特点 |

|------|------|------|

| Antigravity | 谷歌 AI IDE | 多 Agent 编排，最适合小白，免费 |

| Claude Code | 终端 CLI | 终端优先，控制精细，开发者友好 |

| Codex CLI | OpenAI 终端工具 | npm install -g codex，需要 ChatGPT Plus |

三选一，哪个顺手用哪个。核心逻辑一样：它们都能帮你分析日志、修复虾、查问题。

"不管虾死多少次，只要你有其中一个在，就能把它无数次救活。"

总结

| 步骤 | 要点 |

|------|------|

| 准备 | 旧电脑 + 游客账号 + Gemini Pro 邮箱 |

| API Key | Google AI Studio → aistudio.google.com |

| 安装 | Antigravity + OpenClaw，选 Gemini 2.0 Flash |

| 部署 | 本地，不要云端 |

| 配置 | SOUL / IDENTITY / TOOLS / SHIELD 四个文件 |

| 修虾 | Antigravity / Claude Code / Codex 三选一 |
