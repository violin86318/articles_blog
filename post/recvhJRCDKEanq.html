DeepSeek V4：不是最强的，可能是最重要的

过去 8 天，我的收藏夹里多出了 3 篇需要读完的技术报告。

4 月 16 日，Anthropic 发 Claude Opus 4.7。

4 月 23 日，OpenAI 发 GPT-5.5。

4 月 24 日，也就是今天，DeepSeek 发 V4 Preview。

如果你也是做 AI 相关工作的，你懂那种感觉。又是一轮 benchmark。又是一轮"谁吊打谁"的通稿。又是一轮朋友圈里的截图。说实话，我上一次看到 Opus 4.7 发布的时候，第一反应不是激动，是累。

但今天 V4 这篇 58 页的技术报告，我真读完了。

读完之后，有一件事我想和你说清楚。

V4 不是这一周最强的模型。

但它可能是最重要的那一个。

为什么这么说，得从一件反常识的事讲起。

一、V4 没在赌"更聪明"

如果你快速翻一下这三家的官方文案，会发现 Anthropic 和 OpenAI 都在讲同一件事。

Anthropic 说 Opus 4.7 更善于长时程编程，能自己验证自己的输出，视觉分辨率提高了 3 倍多。

OpenAI 说 GPT-5.5 是"一种新的智能类型"，能接着复杂指令一路干到完成，在 Terminal-Bench 2.0 上拿到 82.7% 的 SOTA。

翻译成大白话：它们都在往"更聪明的工作伙伴"这个方向走。

更强的编程。更可靠的 agent。更复杂的工具使用。

价格也跟着往上走。Opus 4.7 和 4.6 同价，$5 / $25 per M tokens。GPT-5.5 涨价了，$5 / $30。GPT-5.5 Pro 直接飙到 $30 / $180。

这是很正统的"旗舰机更新"路线。每一代更强、更贵、更多护栏。

然后 DeepSeek V4 来了。

V4 Pro 的输出价格是每百万 token $3.48。

V4 Flash 的输出价格是每百万 token $0.28。

我把这几个数字放在一起算了一下。

V4 Pro 的输出价格，是 GPT-5.5 的 11.6%。

V4 Flash 的输出价格，是 GPT-5.5 的 0.93%。

按输出 token 算，Flash 比 GPT-5.5 便宜约 100 倍。

严格说还要看口径。按输入 token 算，Flash 比 GPT-5.5 便宜"只有"约 36 倍。但不管怎么比，这不是"便宜几倍"的量级。这是便宜一个到两个数量级。

过去一年我已经习惯了"开源比闭源便宜几倍"的叙事。但"便宜一到两个数量级，而且能力离前沿不远"这种事，我还是第一次见。

所以我打开 58 页技术报告的时候，心里真正的问题不是"V4 能不能打赢 Opus 4.7"，而是：它到底怎么做到的？

读完我发现，V4 根本没在和 GPT-5.5、Opus 4.7 打同一场仗。

V4 在赌一件别的事。

二、V4 赌的是什么

过去一年 AI 应用里真正撞墙的事情，几乎都撞在同一堵墙上。

写代码的 agent 要能读懂整个代码仓库。

法律应用要能一次性吞下几百页合同。

研究助手要能同时看几十篇论文交叉比对。

企业客服要能把整个产品文档、常见问题、历史工单全放进一个 prompt。

这些需求都指向同一件事：上下文要长，而且长得起。

你可能已经听过这件事，但我想停一下，把"长上下文贵在哪里"讲得具体一点，因为这是理解 V4 所有设计的起点。

大模型处理文本的核心机制叫 attention。粗略地说，就是模型每读到一个新词，都要回头把之前所有词再"看"一遍，判断哪些和当前这个词有关系。

问题来了。

假设上下文里有 1000 个词。每个词都要看所有词一遍，总共是 1000 × 1000，也就是 100 万次运算。

现在把上下文拉到 100 万个词。每个词还是要看所有词一遍，但"所有词"现在是 100 万个。

所以运算量变成 100 万 × 100 万。

算出来是多少？

1 万亿次运算。

我知道"100 万 × 100 万 = 1 万亿"这个数字一下反应不过来，因为"100 万"听起来就已经很大了。但仔细展开：100 万是 10 的 6 次方，100 万 × 100 万 就是 10 的 12 次方，12 个零，中文里正好叫 1 万亿。

所以这里的真相是：上下文长度只翻了 1000 倍，运算量却翻了 100 万倍。

这就是 attention 的平方复杂度。每次你把上下文拉长，计算成本不是线性增长，而是按平方爆炸。

还不止计算。每一个词模型都要存一份东西叫 KV cache，你可以粗略理解为"这个词的备忘录"。上下文越长，备忘录越多，显存越吃紧。跑一次 1M token 的推理，光是 KV cache 就能吃掉几十 GB 显存。

所以你会注意到一个规律。最近一年各大厂都在说自己支持 1M 上下文。但实际用起来会发现两件事：一是贵，按 token 算钱的时候上下文拉长账单马上飞起来；二是慢，第一次把长 prompt 喂进去要等好久。

"支持 1M"和"真的能天天用 1M"是两回事。

DeepSeek V4 赌的，就是把后者变成现实。

它赌长上下文是下一个阶段大模型的真战场。

它赌如果能把 1M 上下文的计算成本和显存成本同时砍掉 90% 左右，那么长上下文会从"实验性的奢侈品"变成"日用的基础设施"。

为了这个赌注，V4 几乎每一项技术选择都围着"长上下文的效率"转。

注意力机制被推倒重做，是为了把计算量砍下来。

低精度训练和存储，是为了把显存砍下来。

后训练的整套范式被换掉，是为了把多个领域专家的能力合并到一个统一模型里，同时避免 mixed RL 容易带来的能力互相干扰。

甚至优化器换了、残差连接改了、还加了两个看起来不起眼的训练稳定性技巧，全都是为了让上面这些激进改动在 33 万亿 token 的训练里不会崩。

最后账算下来是这样的：

在 1M token 上下文下，V4 Pro 生成每个 token 需要的计算量只有上一代 V3.2 的 27%，KV cache 只有 10%。

V4 Flash 更极致：计算量 10%，KV cache 7%。

图片：来源内容未完整抓取

这是什么意思？

意思是长上下文里最贵的那两块成本——计算和显存——都被砍掉了八到九成。这不代表你的 API 账单会严格变成原来的十分之一，因为真实账单还和部署、批处理、其他优化有关。但它解释了为什么 DeepSeek 敢把 V4 的价格打到现在这个水平。

这就是 V4 真正的赌注。

它不在赌"我们的模型更聪明"。它在赌"百万上下文的成本曲线会被改写"。

而且 24 小时里我看到的第一批独立评测，已经让人觉得这个赌注赢了一半。Artificial Analysis 把 V4 Pro 列为他们 GDPval-AA 真实工作任务测试里"开源模型第一名"。Simon Willison 在他的博客里只用了一句话总结 V4，翻译过来是"几乎到达前沿，但只要一小部分价格"。

讲到这里你可能在想，这些都是 DeepSeek 自己放的数字，可信吗？

我接下来几节会把他们到底怎么做到的一层一层拆开给你看。架构怎么改，训练怎么稳住，后训练怎么合并，价格怎么压下来。

但在此之前，我想先把一件事说在前面。

DeepSeek 在这次发布里的坦诚，是我最近看过的大模型发布里最少见的。

他们在今天发布的官方公告里直接写了一段话，被多家中文媒体完整引用了。原文大致是：DeepSeek V4 已经是公司内部员工使用的 agentic coding 模型，据评测反馈使用体验优于 Claude Sonnet 4.5，交付质量接近 Opus 4.6 非思考模式，但仍与 Opus 4.6 思考模式存在一定差距。

注意这是 DeepSeek 自己写进发布公告里的话。不是被人追问出来的。他们是主动告诉你：我们没打过 Opus 4.6 思考模式。

他们在技术报告的评测结论里也明确写：V4 比前沿闭源模型"落后大约 3 到 6 个月"。

他们在报告的"不足"章节里主动列出了 V4 在指令精确执行、长文摘要、PPT 视觉美观度上的短板。

这种自报家丑的姿态，在今天的大模型行业里真的罕见。也正是这种坦诚让我觉得 V4 的其他数字更可信。

因为你知道他们没在藏什么。

接下来，让我们看看他们到底怎么做到的。

三、V4 最狠的一刀，砍在注意力上

讲 V4 的技术，必须从它的注意力机制讲起。

这是 V4 所有效率收益里最大的一块。

也是整个模型最激进的改动。

我前面说过，attention 的核心动作是"每读一个新词，回头把之前所有词再看一遍"。这件事的计算量随上下文长度平方增长，是长上下文里最贵的一块。

过去几年大家都在想办法绕过这个平方。最朴素的办法叫滑窗 attention，只看最近的一段，比如只看最近 512 个词。缺点是远距离信息就断了。

稍微聪明一点的办法叫稀疏 attention。在所有历史词里挑出几百个最相关的，只看这几百个。DeepSeek 自家 V3.2 就试过这个路线，叫 DSA，DeepSeek Sparse Attention。

V4 把这个思路又往前推了一大步。

它没有用一种注意力，而是同时用了两种，在不同的层里交替着用。

第一种叫 CSA，Compressed Sparse Attention，压缩稀疏注意力。

第二种叫 HCA，Heavily Compressed Attention，重压缩注意力。

图片：来源内容未完整抓取

我用一个画面来让你理解它俩的区别。

假设你面前有一本 1000 页的书，你要在第 500 页写一个问题的答案，这个答案的线索可能散落在前面 499 页里。

传统 attention 让你把前面 499 页每一个字都再读一遍。非常准，非常贵。

CSA 的做法是：你先把每 4 页压缩成一份小结，总共变成 125 份小结。然后你在这 125 份小结里用荧光笔画出和你的问题最相关的那一批，只读画过荧光笔的那部分。

具体到 V4 的参数上，CSA 把每 4 个 token 的 KV 压成 1 个压缩条目，再通过一个小模块算出重要性分数，挑出最相关的 512 个（V4 Flash）或 1024 个（V4 Pro）压缩条目，只对这些做 attention。

HCA 更激进。它直接把每 128 个 token 的 KV 压成 1 个。不挑了，全读。

还是那本书的比方。HCA 相当于你把整本书按章节浓缩成一句话的章节提要，每章只留一句。然后你每次都把这些章节提要全部读一遍。

压缩比 128 倍。牺牲了细节，但保住了全局视野。

V4 的设计是把这两种注意力交替着排在不同层里。

这样每一层都在做不同粒度的工作。CSA 层抓精细的长距离依赖，HCA 层以极低成本维持全局感知。两种粒度在层与层之间来回切换。

这就是 V4 能把 1M 上下文的单 token 计算量压到 V3.2 的 27%、KV cache 压到 10% 的主要来源。

这里我想停下来说一件事。

稀疏和压缩听起来都很聪明，但只要这种激进的设计有一个小漏洞，模型就会在长上下文上"瞎掉"：关键信息被压没了，或者被稀疏选择漏掉了。

所以 V4 在 CSA 和 HCA 外面还加了好几层防护。

它给每个 query 额外挂了一个 sliding window 分支，让每个新词至少能看到最近 128 个原原本本没被压缩的词。这是保底的近距离细节。

它在注意力里引入了一个叫 sink 的小机制。每个注意力头有一个可学的"出口"，允许 query 把注意力"倒"进去而不是强行分配到某个历史词上。这样模型在不需要强关联的位置可以"算了不看"，避免生出假的强连接。

它还对 query 和 KV 都加了 RMSNorm，防止注意力分数出现数值爆炸。

这些都是细节。但长上下文的注意力设计，胜负就在细节里。

如果你问 V4 相比 V3.2 最大的变化是什么，答案不是参数更多，也不是数据更多。

是注意力从"一刀切"变成了"交替切"。

是从"怎么让 attention 看得更远"变成"怎么让 attention 看得更聪明"。

四、为了敢这么激进，它得稳得住

讲到这里有一个问题会冒出来。

把注意力改成这么激进的稀疏加压缩，真的不会在训练中出事吗？

大模型训练最怕的就是"loss spike"，翻译过来叫"损失函数突然飞起来"。你可能跑了 10 万亿 token 一切正常，某一个 step 突然 loss 从 2.1 跳到 6.8，后面就再也收不回来了。整个 run 报废，几十万美金的算力打水漂。

稀疏 attention、极端压缩、万亿参数 MoE，每一项单独做都会增加 loss spike 的风险。V4 同时做了三项。

所以 V4 的训练稳定性工作，其实比它的架构创新更值得讲。

它不是靠某一个明星级的突破，而是靠好几件事叠在一起。

第一件事是换了优化器。

V4 把大部分参数从传统的 AdamW 换成了一个叫 Muon 的新优化器。

优化器你可以理解为"教练"。模型训练就是教练一遍遍告诉参数该往哪个方向调。AdamW 是过去几年所有大模型的默认教练，稳，但它在训练超大矩阵的时候有一个局限：它给每个参数单独算学习率，结果参数之间的更新方向可能会"打架"。

Muon 的做法不一样。它在每次更新前先做一个数学处理叫"正交化"，让要更新的方向互相不打架，大小也更均匀。

打个比方。AdamW 像是让你每块肌肉单独练，每组次数都由肌肉自己说了算，结果有的肌肉练过头了有的不够。Muon 像是先给你一个整体训练方向，然后强制每组次数保持均匀，整个身体一起往前走。

Muon 不是 DeepSeek 发明的。它来自研究员 Keller Jordan 2024 年发表的一个工作，后来 Kimi 团队的 Moonlight 模型证明它能扩展到 LLM 训练。Kimi K2 用了它。现在 DeepSeek V4 也用了它。

但 DeepSeek 没有一刀切。Embedding、预测头、RMSNorm 这些敏感的小部件还是留给 AdamW。这是一种相当谨慎的折中。

第二件事是改了残差连接。

残差连接是每一层 Transformer 里都有的东西。它的作用是让信号能从底层直通到顶层，不会在深网络里衰减消失。它的公式简单到一行就写完：当前层的输出等于 f(x) + x。

这个设计用了快十年，没人动过。

V4 动了。

V4 用的叫 mHC（Manifold-Constrained Hyper-Connections，流形约束的超连接）。

它做了两步改动。

第一步，把原来一条残差流扩展成 4 条并行的。

第二步，这 4 条流之间要互相混合，但这个混合矩阵必须满足一个严格的数学约束：每一行每一列都加起来等于 1，而且所有元素非负。

这个约束有个名字叫"双随机矩阵"。你不用记住这个名字。你只需要知道它保证了一件事：不管 4 条残差流怎么混合，信号总量永远不会被放大。

这就解决了"多路残差"在实际训练里的老毛病：层数一深，信号经过反复混合就会爆表或者消失。V4 用这个数学约束强行把信号限制在"不扩张"的范围内。

为了每次都让混合矩阵满足这个约束，V4 用了一种叫 Sinkhorn-Knopp 的迭代算法把矩阵投影回合法范围，每次迭代 20 次。

听起来很麻烦。但 DeepSeek 的工程师靠融合 kernel、重计算、流水线调整这一整套手段，把 mHC 的额外开销控制在 6.7% 以内（严格说，这 6.7% 是指它在训练流水线的关键阶段带来的墙钟时间开销，不是整体训练时间的 6.7%，但在工程意义上接近）。

这是一个我读到觉得相当漂亮的工程数字。4 倍宽的残差流不是免费的，但在 DeepSeek 的实现里，代价被压到了一个相当克制的范围。

第三件事是两个看起来"不体面"的小技巧。

第一个叫 Anticipatory Routing，前瞻性路由。

MoE 模型最怕 loss spike 的原因是：专家路由是个小神经网络，主网络一抖，路由也跟着抖，被路由到的专家又生出更大的抖动，恶性循环。

V4 的解法很朴素。让路由用"几步之前的参数"来决定，而不是当下的参数。也就是说，虽然主网络在用最新参数算特征，但"这个 token 分给哪个专家"是用几步前的参数提前算好的。

这样主网络和路由决策就解耦了。主网络抖一下，路由不会立刻跟着抖。

更聪明的地方是：这个机制不是一直开着。V4 只在检测到 loss spike 的时候才启用它，spike 过去就关掉。

DeepSeek 在论文里很难得地写了一句话。翻译过来是：这个方法经验上有效，但我们还不完全理解底层机制。

这种坦诚在今天的论文里真的少见。

第二个叫 SwiGLU Clamping。

SwiGLU 是一种激活函数。Clamping 就是把它的输出强行"夹"在一个范围内。V4 直接把 SwiGLU 的线性分量夹在 -10 到 10 之间，门控分量上限也设 10。

没有理论推导。没有数学证明。就是粗暴地把数值压住。

有效。

你可能会觉得这个朴素得有点"作弊"。但这就是大模型训练的真相：你需要的不是漂亮的理论，是能让训练跑完的所有手段。

把这三件事放在一起看，你会发现 V4 的稳定性工作没有单独一个明星创新。

但这些小心翼翼的组合，让 DeepSeek 敢在 33 万亿 token 上训 1.6 万亿参数的 MoE，敢把 attention 改成激进的混合稀疏。

这是为什么我读这篇报告的时候感觉特别扎实：V4 的每一个激进选择，背后都有一个保守的兜底。

五、把每一个比特都榨干

前面两节讲的是"怎么让训练跑完"。

这一节讲的是"怎么让训练和推理跑便宜"。

关键词是低精度。

先说个背景。大模型里每一个参数，本质上都是一个数字。这个数字用多少位来表示，直接决定了你要花多少显存、多少内存带宽、多少计算。

过去几年的趋势是这样的。

2019 年以前，主流是 FP32，每个数字用 32 位。

2020 到 2022 年，主流变成 FP16 或 BF16，16 位。显存减半，速度翻倍。

2023 到 2024 年，FP8 开始被严肃讨论。再减半。

2025 年开始，FP4 出现在前沿实验室的论文里。再减半。

每降一半，训练和推理的内存、带宽、计算都大幅下降。

但降精度是有代价的。数字能表达的范围变窄，训练容易不稳，推理容易掉分。

所以关键从来不是"用多低的精度"，而是**"在哪里用多低的精度，用怎样的办法让它不掉分"**。

V4 在这件事上的组合很漂亮。

它的 MoE 专家参数用 FP4。其他大部分参数用 FP8。KV cache 里的大部分维度用 FP8，但 RoPE 位置编码相关的几个维度保持 BF16。Lightning Indexer 里的注意力计算甚至直接用 FP4 做。训练的时候还保留着一套 FP32 的主权重，用于梯度累积——不是所有非专家参数都简单变成 FP8。

这些罗列出来就是一堆数字。但里面有一个真正聪明的点，我想单独拎出来讲。

DeepSeek 在报告里有这样一句话：我们的 FP4 到 FP8 反量化是无损的。

这句话要讲清楚，因为很容易被误解。

它说的不是"FP4 可以无损还原成原始的 FP32 权重"。权重在量化成 FP4 那一步已经损失了精度，这个损失回不来。

它说的是另一件事：一旦权重已经被压成 FP4 存起来，再从 FP4 转成 FP8 参与计算的时候，不会再引入第二次损失。

为什么会这样？

FP8 有一种常用格式叫 E4M3，4 位指数、3 位尾数。FP4 常用的 E2M1 格式只有 2 位指数、1 位尾数。FP8 多出来的 2 位指数让它能表达的数值范围比 FP4 大得多。DeepSeek 的发现是：在他们训练时用的 scale 因子约束下，每小块 FP4 权重附带的 scale 信息可以精确地塞进 FP8 的动态范围里，数学上一比一搬过去，不额外损失。

这一座桥听起来小，但工程意义很大。

训练时，你可以用 FP4 存权重省显存，计算时临时转成 FP8 直接接到现成的 FP8 训练框架里，工具链不用大改。梯度继续正常传到 FP32 主权重上。

推理时，直接用 FP4 权重上线，显存和带宽都省了。

这座桥让 V4 同时享受 FP4 的体积优势和 FP8 的生态成熟。

但低精度只是基础设施故事里的一小部分。V4 在工程层面还做了一堆事。

他们写了一套叫 MegaMoE 的 MoE 融合 kernel，把原本分散的通信、计算、激活合并步骤全部合到一个 pipeline 里。这套 kernel 的理论加速上限是 1.92 倍，实测在一般推理场景比非融合的基线快 1.5 到 1.7 倍，在延迟敏感场景（比如强化学习的 rollout 和高速 agent 服务）最快接近 2 倍。这个 kernel 已经在 NVIDIA GPU 和华为 Ascend NPU 两个平台上都验证过，并且以 MegaMoE2 的名字作为 DeepGEMM 的一部分开源了。

他们用了一种叫 TileLang 的 DSL 来写高性能 kernel。用它写出来的代码接近手写 CUDA 的性能，同时能做到比特级的可复现。

他们做了一整套 batch 无关、确定性 kernel。意思是不管 batch size 怎么变、你在哪个时间点跑，模型的输出在比特级别都一致。这件事对调试、对齐训练和推理、对比生产和测试环境都极其重要。

他们重新设计了 KV cache 的存储结构，还做了磁盘级别的 KV cache。意思是如果两个请求共享一段相同的前缀，那段前缀的 KV 可以直接从磁盘读回来，不用重新算一遍。

我不打算把这些全部展开。我只想让你看到一件事。

V4 技术报告里，架构部分大概占了 10 页。基础设施部分占了 9 页。

这个比例在大模型论文里非常罕见。

今天大家发论文都爱讲架构创新，基础设施一般塞在附录里。但 V4 这里，基础设施是它能成立的另一半前提。

MegaMoE2 让 MoE 的通信、计算、激活和合并步骤流水线化，避免专家并行变成新的瓶颈。

mHC 的额外开销靠的是另一套东西：融合 kernel、选择性重计算、流水线调度。两件事不是同一件事。

FP4 和 FP8 之间的无损桥，让 V4 能用最小的显存存权重，同时复用现成的 FP8 训练推理框架。

确定性 kernel 让训练、后训练、推理之间能对齐到比特一致，这对调试和复现都非常重要。

磁盘 KV 缓存让 1M 上下文在有公共前缀的大量请求下变得真正经济。

这些工程选择叠在一起，才是 V4 能把 1M 上下文价格打到 3.48 美元的底气。

V4 的每一项技术创新背后，都蹲着一项工程创新。

六、后训练：先让专家专精，再把他们合成一个

到这里讲的全是预训练。

但一个真正能用的模型，不是预训练跑完就完事了。

预训练结束你拿到的是一个"只会预测下一个 token"的怪物。它能续写，但不会听指令。它能背知识，但不会好好回答问题。它有极强的原始智能，但还没被组织成具体能力。

把它训练成能写代码、能做数学、能当 agent 的模型，靠的是后训练。

这是过去两年 LLM 圈的一个主战场。SFT、RLHF、DPO、GRPO，各种方法五花八门。

V4 在这个战场上做了一次路线调整。

它把 V3.2 里用的"混合 RL"这一大阶段整个换掉，换成了一个叫 OPD（On-Policy Distillation，在线策略蒸馏）的方法。

这个调整背后的思路我觉得值得讲讲。

传统混合 RL 的做法大概是这样：你把不同能力（数学、代码、推理、指令遵循）的奖励信号混在一起，让一个模型同时学这些东西。

好处是简单。坏处是不同能力会互相干扰。数学能力的优化方向可能把代码能力推偏。agent 的奖励信号可能让指令遵循变糟。

V4 的做法是先分开，再合并。

图片：来源内容未完整抓取

第一步，分别训练多个专家模型。一个数学专家，一个代码专家，一个 agent 专家，一个指令遵循专家，等等。每个专家只干一件事，但要做到极致。流程是 SFT 打底，然后用 GRPO（DeepSeek 自家之前在 R1 里用过的 RL 算法）再往上推。

第二步，用 OPD 把这些专家合并到一个模型里。

OPD 怎么合并？学生模型（也就是最终要发布的 V4）自己生成回答。对学生生成的每一个 token，对应领域的专家老师给出自己的概率分布。学生要做的，是让自己的分布去对齐老师的分布。

打个比方。

传统混合 RL 像是把多个老师一起丢进教室里，让他们吵出一个共识，然后学生听这个被吵出来的共识。老师之间的分歧会相互抵消，学生学到的是一个模糊的平均值。

OPD 像是每个老师先单独把自己的课备到顶尖，然后学生挨个去找老师上课，每次上课时老师直接把答案的概率分布摊在桌上给学生看。学生学到的是每个老师各自的巅峰状态。

OPD 在 V4 上有一个关键细节：全词表蒸馏。

什么意思？学生学老师的时候，不是只看老师选的那一个"正确答案"，而是看老师在整个 128000 个可选 token 上的完整概率分布。学生要去匹配这整张分布。

这听起来是理所当然的。但实际上大部分蒸馏方法都做不到，因为存储和计算代价太大。128000 个 token 的 logits，每个位置都要存一份，整个训练过程要存的数据量是天文数字。

大部分方法因此偷懒，只做 token 级别的近似。代价是训练信号方差大，容易不稳。

DeepSeek 为了做全词表蒸馏，专门写了一整套工程设施。他们不存 logits，只存老师最后一层的 hidden state。训练时再用 hidden state 过一下对应老师的预测头，临时重构 logits。存储代价从"128000 乘以序列长度乘以样本数"降到"hidden size 乘以序列长度乘以样本数"。hidden size 一般是 7000 到 14000，比 128000 小一个数量级。

他们甚至专门写了个调度器，把同一个老师负责的样本打包处理，减少模型切换开销。

这种工程投入的目的只有一个：让"理论上正确但做不到"变成"实际能跑而且代价可控"。

最后的结果是，V4 在后训练阶段合并了 10 个以上的领域专家，得到一个在单一模型里同时具备强数学、强代码、强 agent、强指令遵循的统一模型。

OPD 不是 DeepSeek 独创的方法。但他们把它推到这个规模并且公开技术细节，这件事本身是个重要信号。

后训练正在从"让一个模型在混合任务上都差不多"走向"先让每个专家都极强，再用蒸馏合并"。

对开源社区来说，这个转变特别重要。不是说任何人都能按 DeepSeek 的规模完整复现——10+ 个领域专家、全词表蒸馏、专用调度器、长上下文 RL 基础设施，这些都不是小团队能随便搭起来的。但"先让专家都极强、再用蒸馏合并"这套方法论本身，是可以被拆开、被小规模复用、被局部实验的。

八、开源和闭源，已经不在打同一场仗

讲到这里，可以回到开头那个判断了。

V4 不是这一周最强的模型。但它可能是最重要的那一个。

为什么重要？

因为它把一件过去没发生过的事做成了。

它把"接近前沿"的能力，做成了可以完整下载、完整部署、完整审计、完整修改的开放权重。用 MIT 许可证。任何人，任何公司，任何研究者都可以拿去用、拿去改、拿去商用。

而且开放权重的这个版本，价格比闭源前沿便宜一到两个数量级。

这件事的意义不是"开源吊打闭源"。

这件事的意义是：开源和闭源，从 V4 开始，可能不在打同一场比赛了。

闭源前沿在往"更强、更贵、更受限"的方向走。

GPT-5.5 Pro 每百万输出 token $180。Opus 4.7 被很多用户抱怨比 4.6 更费 token。Anthropic 还悄悄保留了一个更强的 Mythos 模型，只给 Project Glasswing 的少数合作伙伴用。OpenAI 的 GPT-5.5 API 还没对所有人开放，要先加额外的网络安全护栏。

这些决定单独看都有道理。前沿能力贵，限制访问有安全理由，高价覆盖高成本合理。

但这些决定加起来画出的是一幅特定的图。

闭源前沿正在变成奢侈品。

开源不追这个方向。

开源在往"接近前沿、更便宜、完全开放"的方向走。

这两条路不是互斥的，但它们服务的场景开始分化。

如果你是一家律所，需要模型在合同里找出一个极其隐蔽的矛盾，你愿意为 Opus 4.7 付钱。

如果你是一家研究机构，需要模型可靠地跑一个 8 小时的 agent 任务，你愿意为 GPT-5.5 Pro 付钱。

但如果你是做 RAG 产品的创业公司，每天要处理几百万份长文档，你会认真考虑 V4 Flash。

如果你是一家欧洲或东南亚的企业，有严格的数据主权要求，V4 是少数几个你真能用的前沿级选项。

如果你是一个 AI 研究者，想在一个接近前沿的开源模型上做实验、做微调、做架构探索，V4 是今天你能拿到的最好选择之一。

如果你是一家中文产品的创业者，V4 Pro 84.4 的中文 SimpleQA 意味着你终于有一个不用把数据送出国、不用付闭源 API 溢价、中文能力明显高于 Opus 4.6 和 GPT-5.4 的开源选项。

这些场景加起来，不是一个小市场。

反过来，我也要把 V4 的真实短板讲清楚。

V4 不是原生多模态。 如果你的产品要处理图片、音频、视频，V4 帮不了你太多。

V4 Pro 在 Hugging Face 上大约 865GB，V4 Flash 大约 160GB（Simon Willison 今天从 HF 实测下载得到的数字）。消费级硬件跑不动 Pro。能本地部署 V4 Pro 的，还是企业用户，或者少数带着 512GB 内存 Mac Studio 和多张 H100 的发烧友。V4 Flash 的本地门槛低一些，但也不是消费级硬件能轻松接住的。

V4 没有 Claude Code 那种打磨到位的开发者工具链。 开源社区会补上，但需要时间。

V4 的长上下文检索质量还没追上 Opus。 1M 上下文的总体积和便宜是 V4 的强项。但"在 1M token 里精确找到某一句"这种任务，Opus 4.6 目前仍然做得更好。

V4 在 agent 任务的一致性和长时程稳定性上落后 GPT-5.5。 尤其是涉及终端操作、电脑使用、多工具并行这些场景。

这些短板是真的。DeepSeek 在报告的"讨论与不足"章节里自己也都列出来了。

所以这是一个不完美的模型。

但它是一个诚实的模型。

它的架构公开。训练方法公开。评测数字和对比方法公开。短板 DeepSeek 自己先说。权重在 MIT 许可证下完整开放。

在一个越来越多东西变成黑盒的行业里，V4 选了一条越来越少人走的路。

这条路不保证它能"赢"。

但它保证了一件事：不管闭源前沿以后走到哪里，开源世界永远有一个可以紧紧咬住它脚后跟的参照系。

这个参照系的存在本身，就在把整个行业的价格曲线往下压。

九、我想用一个画面结束

写到这里，我想用一个画面把整件事收拢。

前沿闭源模型像是智能手机的顶配旗舰。每一代更强、屏幕更好、芯片更猛、AI 更聪明、价格更贵。旗舰机很重要，它们推着技术边界走，也定义了大众对"先进"的想象。

V4 不是旗舰。

V4 更像基础设施。

它做的是铺光纤、建基站、修数据中心这种事。这些事不性感，没有发布会的欢呼，也没有"我们是最强"的通稿。但你每天用的每一个 app，每一个服务，最终都跑在这些基础设施上。

真正改变一个时代的，往往不是最炫的旗舰。而是那一年有多少人开始能用得起宽带，那一年有多少人开始能随身带一个联网设备。

大模型正在走进这个阶段。

旗舰之间互相竞争，benchmark 每月刷新，那是过去两年的故事。下一章的故事不是"哪一家旗舰最强"，而是"接近前沿的能力变成基础设施"。

V4 是这个下一章的第一个信号。

它不是终点。它有明显短板。它自己承认落后前沿 3 到 6 个月。

但它做成了一件具体的事。

它把百万上下文从一个实验性的奢侈品，变成了任何认真做 AI 产品的人都能负担的日用品。

它把整套东西放进了 MIT 开源协议里。

它把 58 页技术报告公开放在 Hugging Face 上给所有人读。

它在 OpenAI 把旗舰价格抬高一倍、Anthropic 用户又开始抱怨 Opus 4.7 比 4.6 更费 token 的那一周，把价格砍到地板上。

这不是一场 benchmark 的胜利。

这是一件更安静、也更深远的事。

在这一周的三个模型发布里，GPT-5.5 和 Opus 4.7 让前沿更强了。

DeepSeek V4 让前沿更便宜了。

这两件事都重要。

但如果你问我未来 5 年哪件事会影响到更多普通人的生活，我的答案是后一件。因为更强的前沿会让少数公司更强。

但更便宜的前沿，会让更多人做出原本做不出的东西。

这才是 V4 今天发布这件事真正的分量。
