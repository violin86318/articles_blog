文章概述

这篇文章介绍了作者开发的一个名为 Web Access 的 Agent Skill，旨在提升 Agent 的联网和浏览器操作能力。该 Skill 能够帮助 Agent 在多个平台上进行操作，如小红书、微博、B站、Boss直聘等，并能自动化地进行网页搜索、内容汇总、发布内容等任务。

文章主要内容

  1. 现有 Agent 联网方案的不足

  2. Claude Code 和 OpenClaw 等 Agent 虽然具备联网工具，但存在一些问题：

    • Claude Code 的模型容易钻牛角尖，无法灵活应对需要登录或动态加载的网站。
    • OpenClaw 虽然支持 CDP 模式，但需要为每个网站单独登录，且并发操作多个网页的能力有限。

  3. Web Access Skill 的特点

  4. 灵活的联网策略：能够根据任务需求选择合适的联网方式，如搜索、静态读取或浏览器操作。

  5. 复用登录态：无需为每个网站单独维护登录状态。
  6. 强大的泛化能力：适应不同网站的联网任务和操作要求。
  7. 支持 Sub-Agent 分治和高并发：能够并行操作多个网页，且不干扰用户浏览器。

  8. 自动沉淀站点操作经验：Agent 在操作网站时，会自动记录操作经验，提高后续任务的执行效率。

  9. Web Access Skill 的设计哲学

  10. 激发模型能力上限的 Skill = Agent 策略哲学 + 最小完备工具集 + 必要的事实说明

  11. Skill 的哲学式设计：提供闭合、高度抽象的思考策略哲学，帮助模型更好地理解任务。
  12. 提供工具的最小完备集：整合了搜索、静态读取、浏览器操作等工具，满足 Agent 的基本需求。

  13. 补充必要的事实说明：强调任务涉及到的惰性知识，提高任务执行的效率。

  14. Web Access Skill 的具体实现

  15. 浏览器选择：使用 Chrome 浏览器及其原生 remote debugging 功能。

  16. 子 Agent 分治：通过 Skill Prompt 设计，鼓励分治，将独立的子任务分配给子 Agent 并行执行。

  17. 经验沉淀机制：每次操作完一个站点，Agent 会自动将该站点的访问策略沉淀下来，下次访问时直接复用。

  18. 未来展望

  19. 在线经验共享池：构想了一个在线经验共享池，让所有用户的 Agent 经验汇聚在一起，形成一个 A2A 社区共建的资产。

  20. Skill 设计经验分享：作者计划在后续的“见知录”系列中分享更完善的 Skill 设计经验。

总结

Web Access Skill 是一个功能强大的 Agent Skill，能够显著提升 Agent 的联网和浏览器操作能力。该 Skill 的设计理念和方法对于其他 Agent Skill 的开发也具有一定的参考价值。
