Hermes 装好了，然后呢？8 个 CLI 动作让 Agent 开干

本文基于 Hermes 官方 CLI 文档和快速入门整理
重点解决：装好以后怎么用？如何让 Agent 真正干活？

昨天我们讲了 Hermes Agent 怎么从安装跑到第一轮对话。

很多人安装完后，就不知道下一步要怎么做。打开 hermes，看到一个终端界面，然后就像普通聊天框一样问问题——问一句、等一句、再问一句。

这样用太浪费了。

今天我们就讲一下，怎么去真正发挥 Hermes 的能力。

💡 核心理念：
Hermes CLI 真正有意思的地方，在于它能不能进入你的工作现场：
✅ 看状态
✅ 调工具
✅ 开后台任务
✅ 恢复会话
✅ 管理上下文

把一个任务从头推进到尾。

1. 不要只问一句

官方 CLI 文档说得很明确：Hermes 的 CLI 是一个完整的终端用户界面（TUI）。

想到deepseek的TUI作者哈哈

image.png：来源内容未完整抓取

⚠️ 重要理解：
不要把 Hermes 理解为网页 UI。它不是只输入一行命令就结束的工具。

它支持：

  • ✅ 多行编辑

  • ✅ 命令补全

  • ✅ 对话历史记录

  • ✅ 中断当前对话

  • ✅ 流式输出

所以特别适合程序员这种长期在终端工作的人。

应该把它当一个可以持续工作的控制台。

举个例子

比如你今天要分析一个项目，不要只问：

这个项目怎么样？ 

❌ 这样问太虚了。

更好的方式是让它先进入现场：

请先浏览当前目录结构，告诉我这个项目的主要模块、入口文件和可能的运行方式。先不要改文件。 

等它看完，再继续问：

基于你刚才看到的结构，帮我列出 3 个最值得先检查的问题，并说明为什么。 

💡 这才像是在带一个 Agent 做事，而不是把它当搜索框。

2. 新手先会这 8 个动作

如果只背一页速查表，我建议先记这 8 个：

02-eight-actions-4k.png：来源内容未完整抓取

① 进入交互界面

hermes 

② 单次查询

hermes chat -q "Hello" 

适合临时问一句，不想进入完整 TUI 的时候用。

③ 看工具

/tools 

你得知道当前 Agent 能不能读文件、能不能跑终端、能不能访问网络，否则很多任务会问得很虚。

④ 切模型

/model 

或者从命令行指定：

hermes chat --model "anthropic/claude-sonnet-4" 

⑤ 看技能

/skills browse 

🔥 重点： Hermes 里安装的 Skills 会自动变成斜杠命令。
这个点很重要，因为它会让通用 Agent 变得更像带工具包的专家。

⑥ 开后台任务

/background Analyze the logs and summarize errors from today 

⑦ 恢复最近会话

hermes --continue 

⑧ 列历史会话

hermes sessions list 

💡 核心总结：
这 8 个动作练熟以后，Hermes 就不再只是“你问它答”。
它会变成一个能挂住工作现场的 Agent 控制台。

3. 一次真实工作流应该长这样

很多人用 Agent 的问题是：任务刚开始，就让它直接给结论。

❌ 这很容易得到一堆空话。

✅ 更稳的方式，是把任务拆成一条链：

03-workflow-4k.png：来源内容未完整抓取

第一步：进入项目目录，然后启动

hermes 

第二步：让它读现场，而不是猜

先浏览当前项目目录，判断这是一个什么类型的项目。只总结结构，不要修改文件。 

第三步：让它拆任务

基于项目结构，给我一个排查计划：先查哪些文件，为什么，预期能发现什么。 

第四步：遇到可以独立跑的分析，开后台

/background 检查当前项目里所有 Markdown 文件，找出标题重复、链接失效和明显格式问题。 

💡 官方 CLI 文档说明：
后台任务会启动一个完全独立的 Agent 会话。
它不阻塞前台，你可以继续聊天、继续推进主任务。

第五步：主会话继续做决策

先不用等后台结果，我们继续看主入口文件。请解释启动流程。 

第六步：退出后恢复

hermes --continue 

🎯 这才是 CLI 实战的感觉：

  • ❌ 不是每次从零开始

  • ❌ 不是问一句断一次

  • ✅ 而是让一个任务有现场、有分支、有恢复

4. 状态栏要看，它不是摆设

Hermes CLI 的状态栏会显示：

  • 模型

  • Token 使用量

  • 上下文条

  • 预估成本

  • 会话时长

官方文档里的示例大概是这种结构：

model │ 12.4K/200K │ [context bar] 6% │ $0.06 │ 15m 

04-status-bar-4k.png：来源内容未完整抓取

上下文颜色含义

官方文档把上下文颜色分成几档：

  • 🟢 绿色：低于 50%，空间充足

  • 🟡 黄色：50% 到 80%，逐渐填满

  • 🟠 橙色：80% 到 95%，接近上限

  • 🔴 红色：95% 以上，接近溢出，建议使用 /compress

⚠️ 这东西别等红了才看。

长任务里，Agent 会读文件、跑工具、总结结果，上下文涨得很快。

查看详细用量

/usage 

上下文压缩

快爆的时候用：

/compress 

💡 官方 CLI 文档说明：
长对话接近限制时会自动摘要，中间轮次会被摘要，而前 3 轮和后 4 轮会保留。

这个设计说明 Hermes 是认真把“长会话工作”当核心场景来做的。

5. 后台任务和会话恢复，是两个完全不同的能力

很多人会把 /background 和 --continue 混在一起。

⚠️ 其实它们解决的是两个问题。

05-background-resume-4k.png：来源内容未完整抓取

/background <prompt> 解决的是并行

比如你在主会话里分析代码结构，同时让后台去检查日志、扫描文件、整理资料。

📌 官方文档说明：
每个后台任务都是独立 Agent 会话，继承当前模型、提供商、工具集和推理设置，但不共享当前对话历史。

这意味着你写后台提示时要给足上下文。

❌ 不要只写：

/background 看一下 

✅ 要写清楚：

/background 分析当前项目下所有 Python 文件，列出可能的安全风险，并按严重程度排序。 

hermes --continue 解决的是续上

比如你今天分析到一半关了终端，明天继续：

hermes --continue 

如果要恢复指定会话，可以用：

hermes --resume  

或者先看：

hermes sessions list 

我建议重要任务一开始就命名：

/title 项目依赖清理 

这样后面找会话不会全靠记忆。

6. 快捷键决定你能不能控住现场

CLI 里最怕什么？

Agent 开始跑以后，你只能盯着它跑完。

Hermes 官方 CLI 文档给了一组关键快捷键：

06-keybindings-4k.png：来源内容未完整抓取

最常用的是这几个。

发送消息：

Enter 

输入多行：

Alt+Enter 

Ctrl+J 

中断当前任务：

Ctrl+C 

退出：

Ctrl+D 

补全斜杠命令：

Tab 

Unix 系统下暂停到后台：

Ctrl+Z 

输入多行这个能力很有用。你可以一次性贴一段需求、一个错误日志、一段代码说明，而不是每行拆开问。

中断也很重要。官方文档说，在 Agent 工作时输入新消息并按 Enter，会中断当前操作并处理新指令；Ctrl+C 也能中断，2 秒内连续按两次可强制退出。

这意味着你发现方向错了，不用等它跑完。

直接改方向。

7. 什么时候用 worktree？

Hermes Agent CLI 教程封面 - 大图版：来源内容未完整抓取

官方 CLI 文档还列了 worktree 模式：

hermes -w 

hermes -w -q "Fix issue #123" 

📌 用途：
隔离 Git worktree，适合并行运行多个代理。

新手不用一上来就开。

但如果你已经在做代码任务，尤其是多个方向同时探索，worktree 会很有用。它能让不同 Agent 在相对隔离的工作区里跑，减少互相影响。

学习路径建议

我会把它放在第二阶段：

  1. ✅ 先熟悉普通 CLI

  2. ✅ 再熟悉后台任务

  3. ✅ 最后再考虑 worktree 并行

把它当控制台。

8 个核心动作

  1. ✅ 用 hermes 进入现场

  2. ✅ 用 /tools 看它能做什么

  3. ✅ 用 /skills browse 找能力
