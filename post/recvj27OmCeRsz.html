TransLit：AI 翻译电子书之我见

一句话总结

译得再慢，也比你读得快——在有限调用速率下，用经济适用型策略实现术语表接力翻译

核心摘要

作者因 Gemini API 配额不足，开始使用 Google AI Studio 调用 Gemini 3.0 Flash 模型，开发了 TransLit 电子书翻译工具。该工具可将 EPUB 内容拆分为 Markdown 片段，分片交给 AI 翻译后合并导出。在开发过程中，作者解决了中文 Markdown 渲染问题（如星号与标点的位置、浏览器缓存等），并重新认识到「术语表」的重要性。通过让每位「译员」在翻译结果后附上新术语，实现了一遍翻译的接力模式，最大程度精简了术语表长度。作者认为虽然翻译速度较慢，但相较于人工阅读仍具优势。

正文整理

作者因 Gemini API 配额不足，开始使用 Google AI Studio 调用 Gemini 3.0 Flash 模型，开发了 TransLit 电子书翻译工具。该工具可将 EPUB 内容拆分为 Markdown 片段，分片交给 AI 翻译后合并导出。在开发过程中，作者解决了中文 Markdown 渲染问题（如星号与标点的位置、浏览器缓存等），并重新认识到「术语表」的重要性。通过让每位「译员」在翻译结果后附上新术语，实现了一遍翻译的接力模式，最大程度精简了术语表长度。作者认为虽然翻译速度较慢，但相较于人工阅读仍具优势。
