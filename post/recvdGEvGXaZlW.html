OpenClaw龙虾技校Skill选课指南

上一篇《Openclaw 龙虾选种及产后护理手册》解决的是“先把虾养活”。这一篇解决的是下一步：龙虾该上学了，课（skill） 怎么选，志愿怎么填🥹。

课选少了，龙虾像白板干不了活；课选多了，龙虾的“课业负担”又太重——启动更慢、上下文更厚、费用更高，skill之间还可能互相冲突，导致龙虾闹情绪不上学😂。所以，问题往往不在单个 skill，而在于课程之间怎么互相搭配、不冲突。所以这篇核心是教大家做一套结构化、负担合理、能真正落地干活的选课方案。

图 1｜课太少像白板，课太多像压垮书包；真正重要的是结构化排课。

一、课表先去哪里查

先说一个最实用的动作：如果你想让龙虾学某个 Skill，找到那个公开 Skill 页，直接把网址发给它，让它去学就行。

另外，文中提到的很多课一些版本的🦞已经作为必修课内置了，不需要再选。此外，很多skill都是个人开发，有一定的使用风险，大家选的时候要注意甄别哦🧐。

关于skil找课，一般有两个主要入口 （本文提到的所有入口和skill都会在文末给出网址）：

  • ClawHub：适合按用途、名字、场景快速找课，比较像选课系统。
  • Awesome OpenClaw Skills：适合系统浏览、批量看分类、找灵感，也更像课程总目录。
龙虾去哪里找课：来源内容未完整抓取

图 2｜找课这件事，既可以走“选课系统”路线，也可以走“总目录”路线。

二、先别急着报热门课，先把基础课修对

基础课看起来不性感，但它决定龙虾后面学什么都不至于变形。像 find-skills、 tavily-search、 summarize、 github、 agent-browser 这类课，本质上是在补搜索、查找、浏览、协作这些底层能力。没有这些，后面的专业课很容易学成花架子。

这里我再推荐两门“元能力”课程： skill-creator 和 skill-vetter。前者代表的是“自己长新能力的能力”，后者更像装课前的教务处：帮你少踩坑、少装重复课、少装互相打架的课。基础课修得稳，后面才不会乱。

龙虾先打基础课地基：来源内容未完整抓取

图 3｜基础课看起来不性感，但它们像地基，决定后面专业课会不会塌。

三、综合素质课不直接发毕业证，但决定龙虾会不会长期工作

有些课看起来不像“具体工种课”，但长期看非常重要。 openclaw-logfire 这种课，解决的是可观测性和排障； self-improving-agent、 proactive-agent、 task-resume 这类课，解决的是自驱、续跑和任务断点恢复； nblm 则很像给龙虾补一门“查过资料再开口”的训练课。

这类课的价值，在于让龙虾不只是会回答，而是会持续工作、会复盘、会纠错。等龙虾真的开始值班，才发现真正稀缺的是稳定性。

四、专业分流学院：同一只龙虾，分院以后才开始像不同工种

龙虾技校九大学院总览：来源内容未完整抓取

图 4｜同一只龙虾，一旦分院，走向的就是完全不同的工种路径。

1）文书与办公室学院

如果你希望龙虾先进入最常见的办公室场景，那最该修的是 ： office、 excel-workflow、 docx、 ppt-ooxml-tool、 powerpoint-pptx、 briefing、 feishu-doc、 boof。

这组课的共同点很朴素：会碰表格、会改文档、会读 PDF、会做 PPT、会把零散信息收成简报。 office 和 powerpoint-pptx 更偏常见办公场景， excel-workflow、 docx、 ppt-ooxml-tool 更偏针对性操作， feishu-doc 适合国内协作环境， briefing 则像把零散材料压成一页纸输出的训练课， boof 可以看作处理文档与文件流的补充课。

2）行政与总助学院

行政和总助岗最值钱的能力，不是“回复得像人”，而是“把碎事收成闭环”。所以这一院我保留 email-autoreply、 coordinate-meeting、 meeting-notes、 meeting-to-action、 feishu-wiki。

邮件、会议、纪要、行动项，这些东西单看都不高级，但它们串起来就是一个真实工位。 email-autoreply 管高频往返， coordinate-meeting 管会务调度， meeting-notes 管记录， meeting-to-action 管收口， feishu-wiki 管长期沉淀。

很多龙虾在聊天时很聪明，一进入行政场景就开始掉链子，问题通常不在理解力，而在没有形成闭环能力。

3）内容与传播学院

内容岗最容易被误解成“会写就行”。其实内容工作的难点从来不是单篇写作，而是持续找题、持续生产、持续分发、持续复用。所以这一院我保留 content-research、 blog-writer、 writing-assistant、 ghost-cms、 blogburst、 blogwatcher、 nblm、 nano-banana-pro。

这套课比较像一条完整链路： blogwatcher 负责盯题源， content-research 负责做资料准备， nblm 负责知识底座， writing-assistant 和 blog-writer 负责成稿， ghost-cms、 blogburst 负责发布与复用， nano-banana-pro 则补足视觉成品能力。

会写一篇不难，能稳定找题、稳定写、稳定发、稳定复用，才算真的进班。

4）编程与研发学院

工程向龙虾最怕的不是不会说，而是只会说。这个方向更重要的是把龙虾放回真实研发现场，而不是让它在群里高谈阔论。所以我保留 github、 openclaw-cursor-codex-connect、 clawd-docs-v2、 subagent-driven-development、 playwright-skill、 mcporter、 tmux。

这一组课里， github 是仓库入口， openclaw-cursor-codex-connect 更偏编辑器/开发链路连接， clawd-docs-v2 管文档查找， subagent-driven-development 负责多人分工式开发， playwright-skill 负责验， mcporter 负责接外部 MCP 工具， tmux 负责交互式 CLI 与远程值班。

真正值钱的工程龙虾，不是解释得多漂亮，而是能不能在正确项目里持续出活。

5）设计与多媒体学院

关于这一院系，我要先说一句实话：设计是贵族专业🥹。 因为图像、音频、视频、3D 这类生成，大多要调外部 API，长期用下来成本不会太低。

如果你确实需要这个方向，可以看 nano-banana-pro、 ComfyUI_Skills_OpenClaw、 agent-media、 ace-music、 3d-model-generation。

这里我做了一个克制处理：不追求全修，而是建议先选一门主修、两门选修。比如多数人先从 nano-banana-pro 开始就够了；如果你更强调流程可控，再加 ComfyUI_Skills_OpenClaw；如果你有音频需求，再补 ace-music；如果你要做更完整的多媒体交付，再看 agent-media；只有确实要碰 3D，再加 3d-model-generation。

这几门课适合把龙虾从“会写字”推进到“会做设计成品”。但如果你只是想尝鲜，别一上来就把视觉、音频、视频、3D 全修满，不然作品没做几个，账单先毕业了。

6）财经与情报学院

如果你养的是值班型、盯盘型、信息流型龙虾，那方向会完全不同。这一院适合 finance-news、 fund-news-summary、 allstock-data、 tradingview-screener、 portfolio-watcher、 nblm。

这一组课更像研究链和跟踪链，不是替你下注，而是帮你把信息收集、筛选、监控、组合观察接起来。 finance-news 和 fund-news-summary 管资讯入口， allstock-data 和 tradingview-screener 管数据与筛选， portfolio-watcher 管长期观察， nblm 则让结论更像“查过资料后的判断”，而不是拍脑袋。

很多人对“投资类 AI”的误解，是它应该直接给买卖答案；但真正有价值的，通常是先把研究和监控做扎实。

7）电商运营学院

电商场景也是一个很实在的方向，核心不在于“说得多会营销”，而在于长期盯竞品、盯价格、盯用户。所以这个方向我保留 competitor-analysis-report、 amazon-competitor-analyzer、 ecommerce-price-watcher、 jtbd-analyzer。

它们解决的事情很具体：看竞品、看价格、看需求、看产品机会。 competitor-analysis-report 更偏通用竞品分析， amazon-competitor-analyzer 更偏平台化场景， ecommerce-price-watcher 负责持续追踪， jtbd-analyzer 则补上“用户到底为什么买”的结构化理解。

8）项目管理学院

项目管理最怕的不是不会想，而是想了很多，事情没往前走。所以这个方向我保留 natural-language-planner、 task-orchestra、 cairn-cli、 task-resume、 meeting-to-action。

natural-language-planner 负责把自然语言需求拆成步骤， task-orchestra 负责多任务编排， cairn-cli 更像项目推进工具课， task-resume 负责断点续跑， meeting-to-action 则补上会议到行动项的收口能力。

这类课不一定让龙虾“看起来最聪明”，但能让它把任务拆开、排开、推进、收口。对很多团队来说，这比多一个会聊天的龙虾重要得多。

9）研究与信息学院

如果你想要的是研究型龙虾，那重点不是多会搜，而是能不能把资料变成判断。所以我保留 nblm、 agent-deep-research、 autonomous-research、 research-report、 truthcheck、 feishu-wiki。

这里的逻辑很清楚：先查，后整，再核，再写。 agent-deep-research 和 autonomous-research 更偏长链研究任务， research-report 负责结构化输出， truthcheck 负责核验， nblm 与 feishu-wiki 则补上知识底座和沉淀结构。会搜索不算本事，能把资料压成结构，最后形成判断，才算研究能力。

五、按龙虾类型排课，别一锅炖

不同类型的龙虾同学：来源内容未完整抓取

图 5｜尝鲜型、文职型、内容型、值班型、工程型、折腾型，气质和课表都不该一样。

尝鲜型龙虾

只配最小课表。基础课里选 2–3 门，再加 1 门轻量工作课就够了。比如 github + blogwatcher + feishu-doc，或者 find-skills + tavily-search + briefing。目标不是毕业，而是先让它动起来。

文职型龙虾

走文书与办公室学院 + 行政与总助学院。重点配 office、 docx、 excel-workflow、 powerpoint-pptx、 meeting-notes、 meeting-to-action。这种龙虾最应该先学会接住真实工位上的碎活。

内容型龙虾

走内容与传播学院，再按预算决定要不要加设计课。核心组合可以是 blogwatcher、 content-research、 writing-assistant、 blog-writer、 nblm、 ghost-cms。如果要往视觉方向走，再加 nano-banana-pro 或 ComfyUI_Skills_OpenClaw。

值班型龙虾

适合财经、研究、项目管理三个方向。重点配 finance-news、 portfolio-watcher、 task-orchestra、 research-report、 truthcheck 这类课。它不一定最会表达，但应该最稳定。

工程型龙虾

走编程与研发学院，重点放在 github、 openclaw-cursor-codex-connect、 playwright-skill、 mcporter、 subagent-driven-development。这类龙虾不缺聪明，缺的是进入真实开发链路的能力。

折腾型龙虾

这种龙虾最容易把自己修废。我的建议是：先定主修，再谈跨院混修。先跑通一条主链，再叠别的课。否则最后就会出现一种很典型的情况：Skill 装了一堆，样样都懂一点，真正交付时谁也不像。

六、最后的校训：先定专业，再选课程

这篇真正想说的，其实很简单。Skill 当然重要，但 Skill 从来不是越多越好。课表臃肿，会让龙虾更慢、更贵，也更混乱。真正成熟的养殖户，不会盯着榜单一门门报热门课，而会先想清楚：我到底想让这只龙虾去做什么。

你想让它当办公室骨干，就修文档、表格、PPT、会议；你想让它做内容，就修选题、写作、发布、复用；你想让它做研究，就修检索、核查、报告；你想让它进研发现场，就把它拉回仓库、文档和协作流里。

龙虾毕业后该走哪条路：来源内容未完整抓取

图 6｜毕业时最重要的，不是所有热门课都修过，而是知道自己要往哪条路走。

所以，课表管理的核心是：不要先问哪个 Skill 最火，先问你想把这只龙虾培养成什么工种。
