先说说为什么要做信息卡图片

最近一年多持续在 X(Twitter) 分享 AI 内容，包括 AI 论文、技术和产品、开源项目等，从最早的 ChatGPT 到 RAG 再到 AI Agent、Coding Agent，从 Prompt Engineering 到 Context Engineering，从 Github Copilot 到 Cursor 再到 Claude Code 等等。

X 账号如果大家感兴趣可以私信沟通哈，不太方便公开发出来，感谢大家的理解。

分享的主要原因最初来自于我自己的学习，让自己必须能跟上 AI 世界最新最全的知识，X 就是一个很好的地方，而以分享为学习方式的「费曼学习法」，也是我特别喜欢的学习方式。

慢慢的分享的内容得到越来越多朋友的关注，因为不确定每个人关注的方向，直接通篇看文字会比较花时间。所以就想做一个信息卡，作为内容的核心内容提炼和展示，让大家可以第一时间通过图片判断这篇内容是否感兴趣，是否要展开读全文、是否要去阅读原论文和开源项目等。

于是就有了信息卡的需求，刚开始是通过 Claude App 来做 SVG 展示，提示词也超级简单：请为内容做可视化展示。

后来自己慢慢就觉得这种形式不满足需求了，机缘巧合看到几位朋友在分享 Bento Grid 这种设计风格，记得有guizang 老师、Orange 老师等等好几位大佬，之前这种风格在 Apple 网页和店里会看到，我自己也很喜欢，所以就试着优化和固化提示词。方法也很原始，就是把 Apple 的网页输入给 AI 模型（我是用 Gemini 2.5 Pro 做实验），让它解读这张图片的设计风格，把提示词帮我总结出来，我拿来生成图片给它看，反复几次之后就能生成接近的设计风格和布局，这个提示词就能固化下来用。

这里也分享一下 BentoGrids 网站，有超多优质设计风格可以参考（一言不合截图给 AI）：https://bentogrids.com/

如果你是产品经理或设计师，这种事做起来会更容易，因为你们能更准确的描述出某种风格叫什么、某种布局是什么，这个名词可以节约很多尝试时间。基于 Bento Grid，我也自己想 + 让 AI 帮我输出常见网页设计布局，给大家看看那咱们最早的六种布局：Bento Grid、卡片、杂志、PPT、视差和互动故事布局。

六种提示词生成六种精美网页布局（Bento、卡片、杂志、PPT、视差和互动故事布局）- 解析纳瓦尔的忠告：停止过度思考
