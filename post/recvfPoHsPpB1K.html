AI 工具 Skill 开源项目 知识管理 约 2 分钟阅读

开源我的所有 skill，以及 skill 管理方法

一句话总结

开源所有技能并基于GitHub CLI实现自动化管理方法，实现跨工具、跨电脑、跨网络的技能共享与定时同步。

核心摘要

作者为实现跨设备、跨工具的skill共享，将100余个自研skill开源并建立系统化管理体系。核心方法包括： 1. 全局skill目录：统一存放于.claude/skills，按写作/画图/信息获取/产品经理四大类分文件夹管理 2. 跨工具同步：通过软链接将Codex等工具的skill根目录同步至全局目录 3. 自动化流程： 使用skill-mannage工具定时自动Git推送 推荐GitHub CLI实现Agent无障碍操作仓库 4. 开源共享：所有skill已同步至GitHub仓库（https://github.com/zephyrwang6/myskill），涵盖AI绘画提示、写作模板等实用技能。 作者强调此体系支持跨网络环境协作，并在社群持续分享skill开发与管理技巧，呼吁读者参与开源生态建设。

正文整理

文章标题为《开源我的所有 skill，以及 skill 管理方法》。文章内容主要讲述了作者在家办公时，为了使自己的电脑也能使用工作电脑的 skill，将 100 多个 skill 进行了同步和分类。这些 skill 可以实现跨工具、跨电脑、跨网络的使用和分享。作者使用的方法包括用 .claude/skills 来做全局 skill，其他工具如 codex 的 skill 根目录做软链接同步 claude/skill，对全局 skill 按类型拆分文件夹，并在 GitHub 建仓同步，最后建立一个 skill-mannage 来定时自动推送 git。作者还强烈建议使用 GitHub CLI 来实现让 Agent 无障碍增删改查 git。文章最后分享了作者的四大类 skill，包括写作类、画图类、信息获取类、产品经理类，并提供了相应的 GitHub 仓库地址。所有 skill 的 GitHub 仓库地址为 https://github.com/zephyrwang6/myskill。作者还在社群里分享了很多关于 skill 工作的创造和管理方法，并鼓励大家点赞支持，后续会分享制作技巧和管理方法，以使 Agent 更有用。
