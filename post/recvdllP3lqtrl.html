正文整理

文章标题为《我让 AI 帮我写公众号，踩了无数坑后，做了一个全自动 Skill》。文章主要讲述了作者如何利用 AI 自动撰写公众号文章的经历，以及在此过程中遇到的种种挑战和解决方案。

文章内容分为以下几个部分：

  1. 引言：作者介绍了自己的背景，以及使用 AI 写公众号的初衷。

  2. 遇到的挑战：

  3. 微信编辑器对样式进行处理，导致粘贴的 HTML 样式丢失。
  4. 微信的 HTML 语法与标准 HTML 有所不同，需要特别处理。
  5. 微信文章必须有封面图，需要通过 API 上传图片并获取 media_id。
  6. 工具链在关键时刻出现兼容性问题。

  7. AI 操作过程中出现错误，导致草稿内容为空。

  8. 解决方案：

  9. 使用微信提供的 API 直接写入 HTML，绕过编辑器。
  10. 遵循微信的 HTML 规则，使用 section 替代 div，用 table 模拟 flex，使用 inline style。
  11. 通过 API 上传图片到微信 CDN，获取 media_id 并保存草稿。
  12. 设计 fallback ladder 降级策略，确保流程的稳定性。
  13. 使用 Pencil 设计模板进行排版和配图。

  14. 生成证据台账，明确信息来源和可信度。

  15. Skill 的设计和实现：

  16. 使用 capability alias 解耦工具，方便更换和升级。
  17. 设计 phase-based pipeline，实现端到端的公众号文章生产。

  18. 支持多种输入和输出格式。

  19. 致谢：作者感谢了翔宇、宝玉和 md2wechat 等人在此过程中的帮助和支持。

  20. 总结：作者表示，尽管 Skill 还不完美，但它已经从幻想变成了现实，每个坑都变成了检查点，每次失败都变成了 fallback ladder 的一级台阶。

  21. 展望：作者希望得到用户的反馈，并询问用户希望他开发什么样的 Skill。

  22. 结尾：作者邀请读者加入 AI 自动化分享群，分享 AI 使用心得和最新动态。
