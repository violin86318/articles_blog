Openclaw与CC的握手，效率翻倍

你可能同时在用 Claude Code 和 OpenClaw。

Claude Code 帮你写代码、改 bug、重构，跑在终端里，专注代码层面。OpenClaw 帮你规划任务、查资料、写文档，跑在桌面上，负责更宏观的协调。

两个工具各有所长，但它们互相不知道对方的存在。你在 Claude Code 里做完一件事，要切回 OpenClaw 手动告知进度。OpenClaw 有个想法，要你自己去 Claude Code 里执行。

来回倒腾，效率损耗全在这里。

Helms-AI MCP Server 就是在这两个工具之间搭一座桥，让它们直接通信。

它是什么

Helms-AI 是一个 MCP Server（Model Context Protocol 服务器）。

MCP 是 Anthropic 推出的一个协议，让 AI 工具能够互相调用对方的能力，就像给 AI 装了个 USB 接口，不同工具之间可以直接插拔数据和指令。

Helms-AI 实现的是：OpenClaw 能直接调用 Claude Code，Claude Code 能直接把结果反馈给 OpenClaw，两边不用人来回传话。

实际效果是：你在 OpenClaw 里描述一个任务，它自动拆解，把代码相关的部分扔给 Claude Code 去执行，执行完结果自动回来，OpenClaw 继续下一步——整个过程你就看着就行。

四个让你眼亮的场景

场景一：写完代码自动通知

Claude Code 改完 bug、跑完测试，直接通过 OpenClaw 发一条 Telegram 消息给你："任务完成，改了 3 个文件，测试全部通过。"

你不用盯着终端，事情完了自然收到通知。

场景二：OpenClaw 指挥 Claude Code 干活

你在 OpenClaw 里说："帮我给这个项目加一个用户权限模块，要符合我们现有的代码风格。"

OpenClaw 拆解任务，把具体的代码实现发给 Claude Code，Claude Code 写完，OpenClaw 拿到结果做代码审查，再决定要不要继续下一步。

两个 AI 分工配合，你就描述了一句话。

场景三：跨工具的上下文共享

以前你在 Claude Code 里积累的项目上下文，OpenClaw 完全不知道。有了 Helms-AI，OpenClaw 可以直接读取 Claude Code 的项目信息，不用你重复解释一遍。

场景四：自动化发布流程

代码写完 → Claude Code 跑测试 → 测试通过通知 OpenClaw → OpenClaw 更新任务状态 → 触发下一步 Lobster 工作流 → 发布完成通知到 Telegram。

整套链路自动跑，你只需要在开头设好规则。

怎么装

第一步：安装 Helms-AI MCP Server

npm install -g helms-ai-mcp

第二步：在 Claude Code 里注册 MCP Server

打开 Claude Code 的配置文件（~/.claude.json），在 mcpServers 里加一条：

{
"mcpServers": {
"helms-ai": {
"command": "helms-ai-mcp",
"args": ["--port", "7788"]
}
}
}

保存后重启 Claude Code，它会自动连接 Helms-AI。

第三步：在 OpenClaw 里启用集成

/integrations enable helms-ai
/integrations configure helms-ai --port 7788

两边都连好之后，确认连接状态：

/integrations status

看到 helms-ai: connected 就说明握手成功了。

第四步：配置通知渠道（可选）

如果你想让完成通知发到 Telegram，配置一下 bot token：

/integrations configure helms-ai --telegram-token YOUR_BOT_TOKEN --chat-id YOUR_CHAT_ID

不需要通知的话跳过这步。

第一次用：让两个 AI 打个招呼

连接好之后，在 OpenClaw 里试一个简单的跨工具任务：

帮我查一下当前项目的 git 状态，用 Claude Code 执行

OpenClaw 会把这个请求转发给 Claude Code，Claude Code 跑完 git status 把结果返回来，你在 OpenClaw 的对话里就能看到结果。

感觉对了之后，就可以开始设计更复杂的协作流程了。
